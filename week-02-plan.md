# Week 2 Plan — AI Solo OS

Date: 2026-05-25
Project Day: Day 8
Status: Approved / Execution Started

## Week 2 Core Goal

Week 2의 핵심 목표는 내부 운영 구조를 더 만드는 것이 아니라, AI Solo OS를 외부에서 실제로 구독하고 받을 수 있는 흐름으로 만드는 것이다.

Week 1에는 프로젝트 기반, 웹사이트, 블로그, Beehiiv, Notion HQ, 운영 DB, 90-Day Checklist v0.1 초안을 만들었다.

Week 2부터는 다음 흐름을 실제로 작동시키는 데 집중한다.

Website → Beehiiv Subscribe → Welcome Email → 90-Day Checklist

## Strategic Direction

AI Solo OS는 Notion 중심의 예쁜 대시보드 프로젝트가 아니다.

Week 2부터는 Google Sheets / Markdown / GitHub 기반의 가벼운 실행 운영 시스템으로 전환한다.

## Tool Roles

| Tool | Role |
|---|---|
| Notion | Day 1~5 아카이브 / 향후 Notion Template 후보 |
| Google Sheets | 실제 운영 DB / KPI / Progress / Handoff / Pipeline |
| Markdown / GitHub | 긴 기록 / 공개 자료 / 체크리스트 원본 / 약관 / 블로그 초안 |
| Website | 공개 허브 / 리드마그넷 제공 / 블로그 |
| Beehiiv | 뉴스레터 / 구독자 커뮤니케이션 / 체크리스트 전달 |
| X | Build-in-Public 배포 채널 |
| Gumroad | Week 2에는 보류 / 이후 상품 판매용 |

## Week 2 Priorities

### Priority 1 — Privacy / Terms 최소 배포

Create and deploy:

- /privacy.html
- /terms.html

Goal:

- 기본 신뢰 확보
- 외부 공개 사이트로서 최소 안내 제공
- 데이터 수집, 제3자 서비스, 연락처, 업데이트 가능성 명시

### Priority 2 — 90-Day Checklist 공개

Recommended structure:

- /checklist.md as source file
- /checklist.html as public page

Public URL target:

https://aisoloos.com/checklist.html

Checklist page should include:

- 제목
- 소개
- 대상 독자
- 90일 원칙
- Day 1~7 체크리스트
- Daily Routine
- KPI 항목
- Beginner blockers
- 다음 단계
- Beehiiv CTA

Checklist page should exclude:

- 내부 운영 회고
- Notion Schema Lock
- Google Sheets 내부 DB 설명
- Product Pipeline 내부 판단
- 팀장 승인 보고 내용

### Priority 3 — Beehiiv Welcome Email 연결

Target flow:

Website CTA → Beehiiv Subscribe → Welcome Email → checklist.html link

Minimum completion criteria:

- Welcome Email 문안 작성
- Checklist link 삽입
- 테스트 구독 또는 테스트 이메일 확인
- 결과를 Google Sheets에 기록

### Priority 4 — Google Sheets 운영 DB 안정화

Google Sheets is the operating DB from Day 7 onward.

Week 2 goal:

Daily records should be enterable in 5–10 minutes.

Avoid over-design.

### Priority 5 — Distribution Rhythm 재개

Week 2 content topics:

- Notion HQ를 만들었지만 Sheets로 옮긴 이유
- 초보 AI 창업자에게 필요한 건 예쁜 대시보드가 아니다
- 무료 체크리스트를 먼저 공개하는 이유
- 구독자 0에서 리드마그넷을 만드는 과정
- Build-in-Public을 팔로워 없이 시작하는 법
- AI Solo OS Week 1 회고
- 90일 실험에서 Day 1~7에 해야 할 일

## Week 2 Prohibited Tasks

Do not work on:

- 유료 상품 출시
- Gumroad 유료 상품 등록
- PDF 디자인
- Notion 템플릿 판매 준비
- Prompt Pack 구성
- Workflow Pack 패키징
- 대규모 랜딩페이지 리디자인
- LinkedIn 확장
- 복잡한 SEO 작업
- Apps Script / 자동화 구축
- 운영 DB 재개편 반복
- 새 운영 DB 추가

## Week 2 Success Criteria

Week 2 is successful if the following are completed:

- privacy.html deployed
- terms.html deployed
- Footer links added
- checklist.html published
- Beehiiv Welcome Email includes checklist link
- Google Sheets operating DB is used daily
- X updates continue
- Week 2 does not drift into overbuilding
